\# 02 - Stakeholders and High-Level Requirements



\## Stakeholder Details



\### Student

\- Register and login

\- Search and enroll in courses

\- View enrolled courses

\- Track payment status



\### Instructor

\- Create and manage courses

\- Upload learning materials

\- View enrolled students



\### Admin

\- Approve instructor accounts

\- Manage course catalog

\- Generate reports

\- Monitor system usage



\### Finance Team

\- Track payment transactions

\- Generate revenue reports



---



\## High-Level Business Requirements



BR-1: The system shall allow students to register and login securely.



BR-2: The system shall provide a searchable course catalog.



BR-3: The system shall allow students to enroll and drop courses.



BR-4: The system shall track course payment status.



BR-5: The system shall allow instructors to create and manage courses.



BR-6: The system shall allow admin to manage users and generate reports.



BR-7: The system shall maintain secure data storage.



BR-8: The system shall support concurrent users without performance issues.



---



\## Non-Functional Requirements



NFR-1: System response time should be under 3 seconds.



NFR-2: System should support at least 500 concurrent users.



NFR-3: All user data must be encrypted.



NFR-4: System uptime should be 99%.
